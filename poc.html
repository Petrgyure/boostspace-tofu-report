4. Make Scenario — Step by Step

⚠️
Blueprint file: make_blueprint.json is in the repo root — importable directly into Make.com via Scenario → Import Blueprint

Module Flow (15 modules)

1
Trigger
Schedule — Every Monday 09:00
Interval: 10080 minutes (1 week). Set start time to Monday 09:00 in Make scheduler.
2
HTTP → POST
PostHog: Reach — Pageviews, Unique Visitors, Sessions
HogQL: SELECT count(), count(DISTINCT person_id), count(DISTINCT $session_id) FROM events WHERE event='$pageview' AND date range
3
HTTP → POST
PostHog: CTA Clicks
Event: cta_click
4
HTTP → POST
PostHog: Install Views
Event: install_view
5
HTTP → POST
PostHog: Pricing Selections
Event: select_pricing_plan
6
HTTP → POST
PostHog: Form Submissions by Type
HogQL GROUP BY properties.form_type — returns rows for sign_up_lead_email, sign_up, demo_lead, callback, waiting_list, newsletter
7
HTTP → POST
PostHog: demo_meeting_booked (MQL)
Critical: if result = 0 for 2+ weeks → alert needed
8
HTTP → POST
PostHog: leady_new_lead
Event: leady_new_lead
9
HTTP → POST
PostHog DW: Google Ads Campaign Stats
Table: googleads_campaign_stats — SUM impressions, clicks, cost_micros/1M, conversions
10
HTTP → POST
PostHog DW: LinkedIn Ads Campaign Stats
Table: linkedinads_campaign_stats — SUM impressions, clicks, landing_page_clicks, cost_in_usd, conversions
11
Set Variable
Assemble weekly_payload JSON
Maps all query results into one week object. Calculates derived metrics (CTA rate, CVR, pages/session).
12
HTTP → GET
GitHub: GET current weekly.json (for SHA)
Required: GitHub API requires current file SHA for update requests. URL: https://api.github.com/repos/Petrgyure/boostspace-tofu-report/contents/data/weekly.json
13
Set Variable
Decode existing weekly.json from base64
GitHub returns file content as base64. Decode → parse JSON → access .weeks array.
14
Set Variable
Append new week + update _meta
New JSON = existing weeks array + new week object. Update _meta.updated_at and next_update.
15
HTTP → PUT
GitHub: PUT updated weekly.json
Base64-encode the new JSON, PUT with SHA from module 12. GitHub Pages auto-rebuilds within ~60 seconds.

6. Dashboard Integration

How the dashboard reads weekly.json

On page load, the dashboard fetches ./data/weekly.json. If successful, it uses the latest week as the "current" column, dynamically updating all KPI cards, table columns and WoW deltas. If fetch fails (offline, CORS), it falls back to hardcoded data.

// On DOMContentLoaded:
fetch('./data/weekly.json')
  .then(r => r.json())
  .then(data => {
    const latest = data.weeks[data.weeks.length - 1];
    // Update KPI cards
    document.getElementById('kpi-pageviews').textContent =
      latest.reach.pageviews.toLocaleString();
    // Update _meta badge
    document.getElementById('last-updated').textContent =
      data._meta.updated_at.substring(0,10);
    // Add new column to all tables...
  })
  .catch(() => console.log('Using hardcoded data'));
✅
Result: Every Monday after Make runs, the dashboard automatically shows the new week without any manual HTML edits. The report stays current with zero effort.
