🔒 Part 2 — Proxy API Route

🔑
Why a proxy?

Right now the PostHog API key is visible in browser DevTools. The proxy is a Vercel serverless function that receives the HogQL query, attaches the secret key server-side, and forwards to PostHog.

ℹ️
File api/posthog.js is already created in the repo. Vercel automatically turns any file in api/ into a serverless endpoint.
// api/posthog.js — already in repo
export default async function handler(req, res) {
  const API_KEY = process.env.POSTHOG_API_KEY;  // from Vercel secret
  const response = await fetch(
    'https://eu.posthog.com/api/projects/100634/query',
    { method: 'POST', headers: { Authorization: `Bearer ${API_KEY}` }, body: JSON.stringify(req.body) }
  );
  res.status(response.status).json(await response.json());
}
✅
Dashboard calls POST /api/posthog → Vercel function → PostHog EU. API key never reaches the browser.

🐳 Part 3 — Docker + Grafana (local visualization)

📊
Grafana OSS — best free data visualization

Runs locally in Docker. Reads weekly.json directly from GitHub raw URL. No database needed. Refresh every hour automatically.

⚠️
Requires Docker Desktop installed. Download at docker.com
1
Install Docker Desktop (if not already installed)
brew install --cask docker
open /Applications/Docker.app
2
Create .env file with your PostHog key
cd /Users/petrgyure/Documents/MCP/tofu_report_preview
cp .env.example .env
# Edit .env and set your actual key:
echo "POSTHOG_API_KEY=phx_R1E6QMDfDUNB8fu7jCjEJ4rgtwHGse2DQaEED88KUaB0mY" >> .env
3
Start the stack
docker compose up -d
# First run downloads images (~500MB). Wait ~60 seconds.
# Grafana installs the Infinity datasource plugin automatically.

docker compose logs -f grafana  # watch startup
4
Open Grafana
open http://localhost:3000
# Login:
#   Username: admin
#   Password: boostspace
✅
Dashboard Boost.space TOFU — Weekly Overview is pre-loaded. 6 KPI stat panels + bar charts + full data table. Reads live from GitHub weekly.json.
5
Stop / restart the stack
docker compose stop    # pause (data preserved)
docker compose start   # resume
docker compose down    # full stop (data preserved in volume)
docker compose down -v # destroy everything including data
